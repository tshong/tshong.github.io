# 開發環境準備

# 1.準備 Eclipse

請建佳準備 eclipse 開發要注意的事項

# 2.安裝 VSCode

  • 參考 Visual Studio Code 下載安裝筆記open in new window
  • 參考影片 VS Code 基本安裝與設置教學open in new window

# 3.安裝本機 Git

  • 參考 Git 介紹與在Windows系統下安裝open in new window

# 4.安裝 Node.js

EZoChamp 開發階段的 server 是前後分離，前端採用 Node.jsopen in new window，後端採用 Spring bootopen in new window。
前後端的運作原理容後介紹。

# 下載與安裝 Node.js

  • 請至官網open in new window 下載 LTS 版本，想進一步了解 Node.js 的同學請自行 google，如這篇open in new window，安裝步驟就跟一般程式一樣，這裡就略過。
  • 除了會用到 Node.js，我們會用到 NPM 模組管理工具，在安裝 Node.js 時會自動裝好 NPM，我們就不必另外裝了。

# 5.下載 EZoChamp 本機包

  • 至中冠 git 下載 EZochamp本機包open in new window

  • 本機包已將所有用到的第三方元件清單寫在build.gradleopen in new window，注意這裡只是清單，不是 jar 檔，你需要執行 gradle 相關程式才會真的從第三方server下載到你本機。

請建佳補充本機執行 build.gradle 下載第三方元件的說明

Line 3: 系統別之後，增加 module 別，這裡的 icsc01 指的是模組代號，請同學用自己的工號替換
Line 4: 為了讓練習資料隔離，請在表格名尾巴掛上自己工號

上述 icsc01 請換成自己的工號

# 2.準備 ezcode

  • 工具目錄: \\\\icscdc01\fileserver\ICSC\software\ezocode\ezcode-0.9.1
  • 將工具放到本機，如: d:\\application\\ezocode
  • 啟動工具: 命令列第 1 個參數為專案路徑的 root ，如: d:\\workspace\\ezochamp-xx

d:\application\ezocode\ezocode.cmd d:\workspace\ezochamp-xx

工具啟動後即進入監聽模式，任何 .dao 有異動，會即時編譯出對應程式碼，不必像過去編寫 .dao 後還要手動執行 codegen，效果如下 :

# 體驗第一個應用

# 登入

啟動 spring boot 後，開啟瀏覽器 http://localhost:8080/ (若你本機已用掉 8080，他會自動改用 8081)
本機的資料庫是連到中冠ERP測試環境，請用中冠測試機自己的工號及密碼登入。

# 進入作業

應用畫面如下，預設會將所有的 primary key 當作搜尋條件，以及所有的欄位都擺在清單上

# 個人化UI

依個人需要，隱藏不需要的欄位

調整欄位寬度

重刷網頁後，仍然會記住剛剛的 UI 調整 記憶的方式是 by 網址、人、電腦。同一台電腦，因為記到個人，所以多人共用一台電腦也不會混亂，但因為個人化調整是記在前端，所以換設備時 UI 得重調。
